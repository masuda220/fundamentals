# ドメインオブジェクト　分析設計パターンカタログ

## 概要

基本的なドメインオブジェクトの分析設計パターンのカタログ（の思索と試作）

## カテゴリ

* 値オブジェクト
	* 数量
	* 金額
	* 日付

* 範囲
	* 数量範囲
	* 金額範囲
	* 日付範囲

* 区分
  * 区分の列挙（名前と序数）
  * 固定データを持つ enum
  * 振る舞いを持つ enum
  * EnumSet
  * EnumMap
  * 区分の階層
  
* コレクション
  * 検査（サイズ、含む）
  * 加工（filter, find)
  * 計算 sum(), average(), ...
  
## 補足

これらを組みあせて、以下のビジネスルールを記述する

* 導出
	* 計算
	* 推論（分類、判定）

* 状態管理（能力管理）
	* 在庫(提供能力)
	* 設備(利用可能)
	* 与信(取引可能)

* 進捗管理と予実差異
	* 約束と実行
	* 契約と履行
	* 計画と実施
	* 予定と実際

* 権限
	* 見える／見えない
	* できる／できない
